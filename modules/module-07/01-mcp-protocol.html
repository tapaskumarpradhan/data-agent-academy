Model Context Protocol (MCP)

The universal standard that lets any AI agent talk to any tool — without custom integration code.

MCP ARCHITECTURE — N+M vs N×M
The Problem MCP Solves

The N × M Integration Nightmare

Maya Chen hit a wall in March. DataPulse — the data agent she had built for AcmeCorp’s 200-store retail operation — worked beautifully with their PostgreSQL warehouse. It could query fact_sales, join across dim_stores and dim_products, and return results in seconds. Then Raj Patel, VP of Analytics, asked her to add Shopify inventory data. Maya wrote a custom connector: bespoke auth, bespoke error handling, bespoke schema mapping. It took her a week.

Then came Salesforce CRM for the sales team. Then Google Sheets, because Derek Kim’s finance group maintained manual revenue adjustments in a spreadsheet. Then the S3 data lake, where historical archives lived. Each connector was bespoke — different auth flows, different error handling, different schemas. “I’ve written five custom integrations,” Maya told Lena Park at their Friday standup. “When Shopify updated their API last month, I spent three days fixing just that one connector.”

Lena did the math. Five data sources, one agent — five integrations. But AcmeCorp was planning three specialized agents for next quarter: DataPulse for analytics, an InsightBot for marketing, and a ComplianceGuard agent that Marcus Webb had been requesting. That meant 3 agents × 5 sources = 15 integrations. “We need a standard,” Lena said. Without one, every new agent or data source multiplied the maintenance burden. That standard is MCP.

Think of it like USB
Before USB, every device had a proprietary cable. MCP is the “USB-C for AI agents” — a single standard that any agent can speak and any tool can implement. Maya’s five bespoke connectors would become five MCP servers, each reusable by every agent AcmeCorp deploys.
Architecture

How MCP Works

MCP uses a client-server architecture built on JSON-RPC 2.0. The agent hosts an MCP client that connects to one or more MCP servers. Each server wraps a specific tool, database, or service. When Maya refactored DataPulse to use MCP, the agent itself shrank — it no longer needed to know how PostgreSQL authentication worked, or how to page through Shopify’s REST API. Each MCP server handled those details internally.

Transport Layer
MCP supports both stdio (local processes) and HTTP + SSE (remote servers). The protocol is transport-agnostic — the same JSON-RPC messages work over either channel. AcmeCorp runs their PostgreSQL MCP server as a local process on the same machine as DataPulse (stdio), while the Shopify server runs as a remote HTTP service shared across all three agents.
Capability Negotiation
On connection, client and server exchange capability declarations. The client discovers what the server can do (tools, resources, prompts) through standardized listing endpoints. When Maya deployed a new Google Sheets MCP server, DataPulse discovered it automatically at startup — no code changes to the agent.
Core Concepts

The Four MCP Primitives

MCP defines four primitive types that cover every interaction pattern between agents and tools. Each one maps to a real need Maya encountered while building AcmeCorp’s integrations.

01
Resources
Data to Read

Resources expose data the agent can read — files, database schemas, API docs, configuration. Think of them as GET endpoints. They provide context without side effects. At AcmeCorp, the PostgreSQL MCP server exposes the star schema (fact_sales, dim_products, dim_stores, dim_dates) as resources so DataPulse can understand the data model before writing queries.

Example: resource://postgres/schema/fact_sales

02
Tools
Functions to Execute

Tools are callable functions with defined inputs and outputs. The LLM decides when to invoke them based on their descriptions. They can have side effects (writing data, sending messages). When Sofia Torres asks DataPulse for Southwest region revenue, the agent calls execute_query on the PostgreSQL MCP server with a generated SQL statement.

Example: tool://postgres/execute_query

03
Prompts
Template Patterns

Prompts are reusable template patterns that servers can offer to clients. They encode best practices for interacting with a specific tool or domain — like pre-built interaction recipes. Priya Sharma, AcmeCorp’s Data Steward, contributed a validate_query prompt template that checks SQL against data governance rules before execution.

Example: prompt://postgres/validate_query

04
Sampling
Server → LLM Requests

Sampling inverts the flow: the server requests LLM completions from the client. This enables agentic behaviors inside the tool server itself — the server can ask the LLM to reason about intermediate results. AcmeCorp’s Salesforce MCP server uses sampling to classify incoming CRM queries (sales pipeline vs. customer support) before routing them to the right Salesforce object.

Example: Server asks LLM to classify a query before executing

Why MCP Matters

Key Benefits

Two months after adopting MCP, Maya presented the results to Raj and Derek Kim. The numbers told the story: what had been 15 planned integrations dropped to 8. Maintenance incidents from API changes fell by 60%. Here is why the protocol delivers those gains.

Portability
Switch LLM providers without rewriting tool integrations. Your MCP servers work with Claude, GPT, Gemini — any MCP-compatible client. AcmeCorp tested a model swap from one provider to another; every MCP server kept working without a single change.
Discovery
Agents dynamically discover available tools and their schemas at runtime. No hardcoded tool registries or manual configuration. When Maya deployed the S3 MCP server on a Thursday afternoon, DataPulse discovered it at its next restart — no redeployment.
Security
MCP servers run in isolation with their own credentials. The agent never sees database passwords — only the MCP interface. Marcus Webb approved the architecture specifically because DataPulse never touches raw PostgreSQL credentials.
Community Ecosystem
A growing library of open-source MCP servers for databases, SaaS tools, cloud services, and more. Build once, share everywhere. Maya found open-source MCP servers for both PostgreSQL and S3 — she only had to write the Shopify and Google Sheets servers from scratch.
Composability
An agent can connect to multiple MCP servers simultaneously. Add capabilities by deploying new servers — no agent code changes needed. DataPulse currently connects to five MCP servers; adding a sixth for a new data source is a configuration change, not an engineering sprint.
Standardized Auth
MCP defines OAuth 2.1 based authorization flows, so authentication is consistent across all tool integrations. No more maintaining five different auth implementations — the pattern Maya hated most about the bespoke connector era.
Deep Dive

MCP Message Lifecycle

Let’s trace what happens when Sofia Torres, Store Manager for Store #47 in the Southwest region, asks DataPulse: “What were my top 10 products by revenue last quarter?” The question passes through four stages of the MCP message lifecycle.

1. Initialize — Client connects and negotiates capabilities
The client sends an initialize request with its protocol version and capabilities. The server responds with its own capabilities (supported primitives, tool list, resource list). Both sides agree on the protocol version to use. At AcmeCorp, this happens once when DataPulse starts up — it establishes connections to all five MCP servers and caches their capability declarations.
2. Discover — Client lists available tools and resources
The client calls tools/list and resources/list to enumerate what the server offers. Each tool includes its name, description, and JSON Schema for parameters. This information is passed to the LLM for tool-use decisions. DataPulse learns that the PostgreSQL server offers execute_query, list_tables, and get_schema tools, along with schema resources for every table in AcmeCorp’s star schema.
3. Invoke — Agent calls a tool via JSON-RPC
When the LLM selects a tool, the client sends a tools/call request with the tool name and arguments. The server executes the function and returns the result. The response includes structured content (text, images, or embedded resources). For Sofia’s query, DataPulse sends an execute_query call with SQL that filters fact_sales by Store #47 and AcmeCorp’s fiscal Q3 (July–September).
4. Respond — Result flows back to the LLM
The tool result is formatted and returned to the LLM as an observation. The LLM can then reason about the result, make another tool call, or compose a final response to the user. The cycle repeats as needed. DataPulse receives Sofia’s top-10 product list, formats it with revenue figures and percentage changes, and delivers the answer — all within seconds.
Case Study

AcmeCorp’s MCP Architecture

By June, Maya had replaced all five bespoke connectors with MCP servers. The before-and-after picture was stark: what had been a fragile web of custom code became a clean, modular architecture. Lena Park called it “the best refactor we’ve done all year.”

Before vs. After
Before MCP: 5 bespoke connectors, each with unique auth code, error handling, and schema mapping. Shopify API v2024-01 update broke DataPulse for 3 days. Maya was the only person who understood all five connectors.

After MCP: 5 standardized MCP servers, each independently deployable and testable. When Shopify updated their API again in May, Maya fixed the Shopify MCP server in 2 hours — DataPulse and the other two planned agents never noticed.
01
PostgreSQL MCP Server
Exposes execute_query tool, schema resources for all 500+ tables in AcmeCorp’s warehouse, and a validate_query prompt contributed by Priya Sharma. Handles connection pooling, credential management, and query timeouts internally.
02
Shopify MCP Server
Wraps Shopify’s REST and GraphQL APIs for inventory and order data. Provides get_inventory and list_orders tools. When Shopify releases a new API version, Maya updates this single server — all agents benefit immediately.
03
Salesforce MCP Server
Connects to AcmeCorp’s Salesforce CRM. Offers query_accounts and get_pipeline tools. Uses sampling to let the LLM classify whether an incoming request is about sales pipeline or customer support before hitting the right Salesforce object.
04
Google Sheets MCP Server
Derek Kim’s finance team maintains manual revenue adjustments in Google Sheets. This server exposes read_range and list_sheets tools so DataPulse can incorporate the finance team’s corrections without anyone changing their workflow.
05
S3 Data Lake MCP Server
AcmeCorp’s historical archives (pre-2023 data) live in S3 as Parquet files. This server provides query_parquet and list_objects tools. When Raj needs year-over-year comparisons, DataPulse seamlessly queries both PostgreSQL (current) and S3 (historical).
Key Insight
Adding a new data source is just deploying a new MCP server. The agent discovers it automatically through capability negotiation. No redeployment of the agent itself required. When AcmeCorp onboards their next data source, Maya estimates it will take days instead of the weeks each bespoke connector used to require.
