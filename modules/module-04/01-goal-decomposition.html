Module 04 — Autonomy & Collaboration

Goal Decomposition

Complex requests require systematic breakdown. A single agent can’t solve everything in one step — it needs a plan.

GOAL DECOMPOSITION — TASK DAG
AcmeCorp Scenario

AcmeCorp’s VP of Operations asks: “Give me a complete Q4 performance analysis with year-over-year comparisons, regional breakdowns, product category trends, and recommendations for Q1.”

One agent can’t handle this in a single step. The request spans multiple data sources, analysis dimensions, and output formats. Time to decompose.

Three Planning Levels

Just like a military operation or a software project, agent planning happens at three levels of abstraction.

01 Strategic

What to Achieve

The high-level business objective. What does success look like?

AcmeCorp

“Understand Q4 sales performance comprehensively”

02 Tactical

How to Get There

Concrete approach — what steps and in what order?

AcmeCorp

Pull sales data → Compare to targets → Analyze by region → Analyze by product → Generate recommendations

03 Operational

Do It

Specific executable actions the agent performs.

AcmeCorp

Run SELECT region, SUM(revenue)... → Calculate YoY% → Build bar chart → Write summary

Task Dependency Graph

The Q4 analysis decomposes into seven tasks. Some can run in parallel (A & B), while others must wait for dependencies.

Goal decomposition works just like planning a road trip. Each level of planning gets more concrete:

Strategic

“Get from New York to Los Angeles”

Tactical

“Route through Chicago, Denver, and Vegas + book hotels at each stop”

NYC→ Chicago→ Denver→ Vegas→ LA
Operational

“Start car, merge onto I-80 West, drive 4 hours, stop for gas at exit 142, refuel at Shell station…”

Why Decomposition Matters

Exceeds Context Windows
A complete Q4 analysis may need millions of tokens of data. Decomposition lets each sub-task operate within context limits.
Better Error Handling
If the regional breakdown fails, retry just that sub-task. No need to re-run the entire analysis from scratch.
Enables Parallelism
Tasks A and B have no dependencies — run them simultaneously. Total time drops from 7 sequential steps to ~4 critical-path steps.
Traceable Reasoning
Every recommendation traces back through a clear chain: recommendation ← analysis ← data query. Auditable and explainable.
