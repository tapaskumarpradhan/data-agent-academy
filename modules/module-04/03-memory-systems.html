Module 04 — Lesson 03

Memory Systems

Memory is what separates a stateful agent from a stateless chatbot. Without it, every question exists in a vacuum.

AGENT MEMORY ARCHITECTURE
The Scene

The Blank Stare

AcmeCorp’s VP of Sales asks the agent “What about tomorrow?” after asking about weather-related sales impact. Without memory, the agent has no idea what “tomorrow” refers to. It can’t connect the dots. Memory is what separates a stateful agent from a stateless chatbot.

AcmeCorp Scenario
The VP just asked about rain’s impact on foot traffic. Seconds later, they follow up: “What about tomorrow?” A stateless chatbot responds: “I’m sorry, can you clarify what you mean?” A memory-enabled agent instantly knows the VP is asking about tomorrow’s weather forecast and its expected sales impact.
Three Memory Layers

How Agents Remember

1 Working Memory

The current context window. What the agent is actively processing right now — the live registers of its thought process.

Current Context
AcmeCorp in Action
The agent holds the current SQL query, the user’s question, and the last 3 tool results in working memory.
Your desk — what you’re actively working on.
2 Session Memory

The current conversation. Conversation history within the current chat session — recent messages, actions, and preferences the user expressed during this interaction.

Session Scope
  • Last N messages from the conversation
  • Recent tool calls and their results
  • User preferences expressed during the session
AcmeCorp in Action
When the VP says “break that down by region,” the agent remembers “that” refers to Q3 revenue they just discussed.
Your notebook — notes from this meeting.
3 Long-Term Memory

Persistent across sessions. Knowledge that survives beyond any single conversation — the agent’s accumulated wisdom about your organization, data, and preferences.

Persistent
  • Factual: “Table X has columns A, B, C”
  • Episodic: “User asked about sales last Tuesday”
  • Procedural: “To compute revenue, use SUM(total_amount)”
  • Preferences: “User prefers monthly views, bar charts over tables”
AcmeCorp in Action
When the VP returns next week, the agent remembers they prefer regional breakdowns and that “revenue” means total_amount excluding returns.
Your filing cabinet — reference material built over years.
Memory Lifecycle

The Consolidation Cycle

Memory doesn’t just happen — it flows through a deliberate consolidation cycle at every session boundary.

A good accountant remembers that Mrs. Johnson prefers email, has a home office, and her fiscal year ends in March. They don’t ask every year. They built that knowledge over dozens of interactions and filed it away. When Mrs. Johnson calls in February, the accountant already knows it’s tax prep time without being told.

Your data agent should do the same. It should learn user preferences, remember domain-specific definitions, and recall past interactions — not start from scratch every time.

The Difference

Without Memory vs. With Memory

Without Memory
User: “What about tomorrow?”

Agent: “I’m sorry, can you clarify what you mean?”
With Memory
User: “What about tomorrow?”

Agent: “Tomorrow’s forecast shows light rain, which historically correlates with a 12% dip in foot traffic at your Northeast stores.”
