AGENT STATE MACHINE
The Scene

More Than Request-Response

AcmeCorp’s agent doesn’t just sit and wait for questions. It operates in different modes — sometimes reactive, sometimes proactive. And when things go wrong, it needs to recover gracefully. A production-grade agent must handle both transitions and failures with intention.

Reactive
Waiting for input. The agent is idle, listening for user queries or system events. It responds to what it receives.
Agent sits idle until the VP asks a question.
Analogy: The Self-Driving Car
A self-driving car cycles through the same states: reactive (stopped at a light), reasoning (reading signs and sensor data), planning (calculating the optimal route), executing (driving the planned route), and proactive (rerouting around traffic it detected ahead). Your data agent follows the same pattern.
Failure Handling

When Things Go Wrong

Production agents encounter failures constantly — timeouts, service outages, rate limits, context overflow. The question isn’t whether failures happen but how the agent recovers.

1 Retry with Backoff

Try again with increasing delay between attempts. Most transient failures (timeouts, rate limits, network blips) resolve themselves if you wait.

Transient Failures
AcmeCorp in Action
Database timeout → wait 1s → retry → wait 2s → retry → succeed. The exponential backoff prevents hammering an already-struggling service.
2 Fallback to Alternative

When the primary approach fails repeatedly, switch to a different data source, tool, or method. A different path may succeed where the first one failed.

Persistent Failures
AcmeCorp in Action
Primary Snowflake warehouse down → fallback to read replica → partial data is better than no data. The agent transparently notes the fallback source.
3 Graceful Degradation

Return what you have, even if it’s incomplete. A partial answer with context is always more valuable than a blank error screen.

Partial Capability
AcmeCorp in Action
Can’t generate chart → return the data table with a note: “Chart generation unavailable, here’s the raw data.” The VP still gets value.

If your GPS loses signal, it doesn’t give up. It uses last known position, estimates based on speed and direction, and recalculates when the signal returns. Your data agent should behave the same way — use cached data, estimate with what it has, and recover when services come back online.

Cascade Prevention

Circuit Breakers & Handoffs

When one agent fails, the system must prevent the failure from cascading. Circuit breakers detect repeated failures and trigger handoffs before the whole system goes down.
