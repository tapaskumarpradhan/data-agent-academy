Module 04 — Autonomy & Collaboration

Multi-Agent Collaboration

One agent isn’t enough. Complex enterprise tasks need teams of specialized agents working together — just like human data teams.

MULTI-AGENT ARCHITECTURE
AcmeCorp Scenario

AcmeCorp’s Q4 analysis needs aren’t getting done fast enough with a single agent. They need a team of agents working together — just like their human data team. By 2026, 72% of enterprise AI projects use multi-agent architectures.

Sequential Handoff

Relay Race Low Complexity

Agents pass work forward in a chain. Each agent completes its task before handing off to the next, like runners in a relay race.

AcmeCorp

Order fulfillment: Inventory Agent checks stock → Pricing Agent applies discounts → Fulfillment Agent generates shipping label.

Best for: Document review, data processing pipelines, any workflow with strict ordering.

Parallel Execution

Potluck Dinner Medium Complexity

A request fans out to multiple agents simultaneously. Each works on its piece independently, then results merge back together.

AcmeCorp

Quarterly analysis: Sales Agent, Inventory Agent, and Customer Agent all analyze their domains simultaneously — results merge into one comprehensive report.

Best for: Independent analyses, ensemble reasoning, any task where sub-problems don’t depend on each other.

Supervisor Pattern

General Contractor High Complexity

A supervisor agent decomposes the task, delegates to specialists, monitors progress, and synthesizes results. The supervisor maintains the big picture while specialists focus on details.

AcmeCorp

AcmeCorp’s Super Agent decomposes “Q4 analysis” → delegates to SQL Agent, Chart Agent, and Report Agent → synthesizes their outputs into one executive-ready deliverable.

Best for: Complex domains with clear specialization, enterprise workflows, coordinated multi-step processes.

Debate Pattern

Jury Deliberation High Complexity

Two or more agents argue different positions. An evaluator agent weighs the evidence and produces a balanced conclusion. Disagreement improves decision quality.

AcmeCorp

When AcmeCorp’s forecasting agents disagree on Q1 predictions (one bullish, one conservative), a mediator agent weighs evidence and produces a balanced forecast.

Best for: Complex decisions where multiple perspectives improve quality, risk assessment, forecasting.

Agent-to-Agent (A2A) Communication

A2A

The Protocol Layer

Think of A2A like diplomats from different countries using a common language to negotiate. Each agent may be built with different frameworks, run on different platforms, or specialize in different domains — but A2A gives them a shared protocol for collaboration.

A2A Protocol

Focuses on agent-to-agent collaboration. How agents discover each other, negotiate capabilities, delegate tasks, and share results.

MCP (Model Context Protocol)

Focuses on tool integration. How agents connect to databases, APIs, file systems, and external services.

Key Insight

A2A and MCP are complementary, not competing. A2A handles the conversation between agents; MCP handles how each agent accesses its tools. Together, they form the full communication stack.
