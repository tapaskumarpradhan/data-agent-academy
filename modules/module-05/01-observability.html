OBSERVABILITY — TRACE ANATOMY
The Scene

The Silent Failure That Nobody Noticed

AcmeCorp’s Product Owner agent had been silently degrading for two weeks. The VP of Sales kept getting answers, but accuracy had dropped from 94% to 71%. Nobody knew because there was no observability dashboard. The agent was confidently returning wrong revenue numbers.

Incident Report
A Shopify schema migration changed the total_amount column precision from INT to DECIMAL. The agent’s SQL was syntactically valid, but the numbers were subtly wrong — rounding errors compounded across thousands of rows. By the time an analyst spot-checked a report, three board decks had been presented with inaccurate data. The CEO asked: “How did we not catch this for two weeks?”

OpenTelemetry

The Emerging Standard

OpenTelemetry (OTel) is becoming the vendor-neutral standard for instrumenting LLM applications. It provides semantic conventions for traces, metrics, and logs — meaning your instrumentation code stays the same regardless of which observability backend you use.

⚡ Why OpenTelemetry Matters

Vendor neutrality. Instrument once, export anywhere. Your agent code emits OTel spans — those spans can flow to Langfuse, Datadog, Jaeger, or any OTel-compatible backend.

Semantic conventions. OTel is developing LLM-specific semantic conventions: gen_ai.system, gen_ai.request.model, gen_ai.usage.input_tokens, gen_ai.usage.output_tokens. These standard attribute names make traces from different teams and tools interoperable.

Future-proof. As the ecosystem consolidates, OTel will be the common wire format. Investing in it now means you won’t have to re-instrument when you switch platforms.

AcmeCorp
By using OpenTelemetry semantic conventions, AcmeCorp can export traces to Langfuse today and switch to Datadog tomorrow without changing agent code. No vendor lock-in.

1. Trace everything. Every agent execution should produce a structured trace with a unique ID. Without traces, debugging production issues is guesswork.

2. Monitor accuracy, not just uptime. An agent can be “up” and returning results while being subtly wrong. Human-rated accuracy sampling is the only way to catch silent degradation.

3. Use three levels of granularity. Trace-level for the big picture, step-level for bottleneck analysis, guardrail events for security and compliance.

4. Invest in OpenTelemetry. Vendor-neutral instrumentation means you can switch backends without rewriting code. The LLM semantic conventions are maturing fast.
