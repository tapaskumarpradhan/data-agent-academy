The Bridge

From Intent to Execution

The LLM has decided it needs sales data. But how does "I want to query the database" become actual SQL execution? This is the bridge between reasoning and action — the tool calling protocol.

Tool calling is the mechanism that turns the LLM's intent into structured function calls. The model doesn't run code — it emits a JSON request, and the runtime executes it. Let's trace this step by step.

The Protocol

Anatomy of a Tool Call

1

User Prompt

U
"Show me Q3 revenue by region"
2

LLM Reasoning

Internal Reasoning
"The user wants revenue data segmented by region for Q3. I need to first discover which tables contain revenue data, then build and execute a query. Let me start with schema discovery..."
3

Function Call Emitted

The LLM outputs a structured JSON request — not code, just a declaration of intent.

llm_output → function_call
{ "tool": "schema_discovery", "parameters": { "search": "revenue sales", "include_columns": true } }
4

Tool Execution Result

The runtime executes the function and returns structured data back to the LLM.

tool_result → llm_context
{ "status": "success", "tables": [ { "name": "sales_transactions", "columns": ["date", "region", "revenue", "store_id"] }, { "name": "quarterly_revenue_summary", "columns": ["quarter", "region", "total_revenue"] } ] }
5

LLM Formats Final Response

After potentially multiple tool-call rounds, the LLM synthesizes a natural-language answer.

A
"Here's the Q3 revenue breakdown by region:

North: $4.2M   South: $3.8M   East: $5.1M   West: $4.6M

The East region leads Q3 revenue, up 12% from Q2. Would you like me to drill into any specific region?"
Two Perspectives

User View vs. System View

The magic of a well-built agent is that the user sees a simple conversation while a complex orchestration happens behind the scenes.

What the User Sees
U
Show me Q3 revenue by region
Thinking...
A
Here's your Q3 revenue by region:
North: $4.2M | South: $3.8M
East: $5.1M | West: $4.6M

East leads, up 12% from Q2.
What Happens Behind the Scenes
1. Parse Intent
Extract: metric=revenue, period=Q3, groupby=region
2. Tool Call: schema_discovery
Search tables matching "revenue" → found 2 candidates
3. Tool Call: execute_query
SELECT region, SUM(revenue) FROM quarterly_revenue_summary WHERE quarter='Q3' GROUP BY region
4. Tool Call: execute_query
SELECT region, SUM(revenue) FROM quarterly_revenue_summary WHERE quarter='Q2' GROUP BY region
5. Synthesize
Compute Q2→Q3 delta, format natural language response
Full Trace

AcmeCorp Worked Example

Let's trace every iteration the agent takes to answer: "Show me Q3 revenue by region"

AcmeCorp Context
AcmeCorp's data warehouse has 47 tables. The agent has access to schema_discovery, execute_query, validate_sql, and format_results tools. The analyst has "regional_viewer" permissions.
Iteration 1: Schema Discovery ›
Iteration 1 schema_discovery

LLM Reasoning: User wants Q3 revenue by region. I need to find which tables have revenue data and a region column.

function_call
{ "tool": "schema_discovery", "parameters": { "search": "revenue region quarterly" } }

Result: Found quarterly_revenue_summary with columns: quarter, region, total_revenue, store_count.

Iteration 2: Build & Validate SQL ›
Iteration 2 validate_sql

LLM Reasoning: I'll build a SELECT query and validate it before execution to catch syntax issues.

function_call
{ "tool": "validate_sql", "parameters": { "query": "SELECT region, SUM(total_revenue) AS revenue FROM quarterly_revenue_summary WHERE quarter = 'Q3-2025' GROUP BY region ORDER BY revenue DESC" } }

Result: Valid. Estimated rows: 4. Estimated time: 120ms.

Iteration 3: Execute Query ›
Iteration 3 execute_query

LLM Reasoning: Validation passed. Now execute the query. I'll also fetch Q2 data for comparison context.

function_call
{ "tool": "execute_query", "parameters": { "query": "SELECT region, SUM(total_revenue) AS revenue FROM quarterly_revenue_summary WHERE quarter IN ('Q2-2025','Q3-2025') GROUP BY region, quarter ORDER BY region" } }

Result: 8 rows returned (4 regions x 2 quarters). Execution time: 95ms.

Iteration 4: Format & Respond ›
Iteration 4 format_results

LLM Reasoning: I have Q3 data and Q2 data for context. I'll compute the quarter-over-quarter change and present it clearly.

function_call
{ "tool": "format_results", "parameters": { "data": [{"region":"East","q3":5100000,"q2":4553571},{"region":"West","q3":4600000,"q2":4371428},{"region":"North","q3":4200000,"q2":4000000},{"region":"South","q3":3800000,"q2":3619047}], "format": "table_with_delta", "title": "Q3 2025 Revenue by Region" } }

Result: Formatted table with Q3 values and QoQ change. The LLM composes the final natural-language answer with key insight (East leads, +12%).
