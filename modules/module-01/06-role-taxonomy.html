Agent Specialization

Role Taxonomy

AcmeCorp realizes one agent isn't enough. Sales needs something different from data engineering. Different problems call for different agent roles.

The Three Roles

Agent Archetypes

1 Orchestrator

Workflow coordinator. Delegates to sub-agents, manages handoffs, and ensures the pipeline completes end to end.

Like a wedding planner — doesn't bake the cake or arrange the flowers, but makes sure everything happens in the right order.

AcmeCorp
Coordinates the pipeline: ingest, transform, validate, then generate the report.
2 Super Agent

Cross-domain generalist. Broad capabilities, can handle ambiguous requests by deciding what analysis is needed.

Like a management consultant — assesses the situation, defines the approach, and marshals resources.

AcmeCorp
Handles "analyze our Q3 performance" — decides what data, what metrics, what format.
3 Utility Agent

Single-purpose specialist. Fast, reliable, does one thing extremely well with minimal overhead.

Like a translator — takes input in one form and converts it to another, consistently and quickly.

AcmeCorp
sql_agent — takes natural language, returns SQL, nothing else.

AcmeCorp in Practice
In practice, AcmeCorp uses a Super Agent as the front door. It routes to Utility Agents for SQL generation, Shopify API calls, and report generation. An Orchestrator manages the nightly data pipeline that feeds them all.
