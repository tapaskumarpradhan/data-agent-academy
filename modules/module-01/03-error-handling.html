The Scene

When Things Go Wrong

It's Monday morning. AcmeCorp's database is under heavy load from weekend batch jobs. The agent starts getting timeouts. An analyst asks about a table that was renamed last week. A store clerk tries to peek at salary data. A product SKU lookup hits a discontinued item.

A production-grade Data Agent doesn't just handle the happy path. It needs a strategy for every failure mode. Let's explore the five error categories your agent must handle, each illustrated with a real AcmeCorp scenario.

Error Taxonomy

Five Error Types Every Agent Must Handle

T Transient Errors

What: Network timeouts, database busy signals, rate limits. The request is valid but the system is temporarily unable to serve it.

Strategy: Exponential Backoff
1s
Retry 1
2s
Retry 2
4s
Retry 3
AcmeCorp Scenario
Database busy from weekend batch jobs. Agent retries with exponential backoff: 1s, 2s, 4s. On the third retry, the batch job completes and the query succeeds.
V Validation Errors

What: Invalid SQL syntax, bad parameters, malformed requests. The user's intent can't be translated to a valid operation.

Strategy: Clear Error + Guidance
validation_error.log
// User: "Show me data from the inventory_v1 table" ERROR: Table "inventory_v1" does not exist. SUGGESTION: Did you mean "inventory_v2" or "inventory_current"?
AcmeCorp Scenario
An analyst asks about a table that was renamed after last month's migration. Instead of a cryptic SQL error, the agent suggests the correct table names.
P Permission Errors

What: Unauthorized access attempts. The user is trying to access data beyond their role's scope.

Strategy: Deny + Explanation
access_control.log
ACCESS DENIED Role: store_clerk | Requested: regional_salary_data // Policy: salary data restricted to HR + Finance roles RESPONSE: "I can't show salary data for your role. Contact HR or your regional manager for access."
AcmeCorp Scenario
A store clerk tries to access regional salary data. The agent explains the access restriction and directs them to the appropriate channel for requesting access.
N Not Found Errors

What: Missing tables, deprecated resources, deleted data. The thing the user wants simply doesn't exist anymore.

Strategy: Suggest Alternatives
resource_lookup.log
NOT FOUND: Product SKU "AC-WIDGET-001" discontinued ALTERNATIVES: - "AC-WIDGET-002" (successor product) - "products_archive" table for historical data - "similar_products" view for related items
AcmeCorp Scenario
A product manager looks up a discontinued SKU. The agent points them to the successor product and the archive table for historical comparison.
O Timeout Errors

What: Long-running queries that exceed the allowed execution window. The query is valid but too expensive to complete in one shot.

Strategy: Partial Results + Continue
query_execution.log
TIMEOUT after 30s | Query: 3-year cross-store join PARTIAL RESULTS: 2 of 3 years completed (2022, 2023) // Offering user a choice: "I have results for 2022-2023. Want me to continue with 2024, or is this enough?"
AcmeCorp Scenario
A finance analyst runs a complex three-year cross-store revenue join. After 30 seconds, the agent returns partial results for 2022-2023 and offers to continue fetching 2024 data.
