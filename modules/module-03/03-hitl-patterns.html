Trust & Control

Human-in-the-Loop
Patterns

Human-in-the-loop (HITL) is not optional for enterprise agents — it’s a regulatory and trust requirement. The question isn’t “should we have HITL?” but “where and how should we insert it?” Getting HITL wrong means either bottlenecking every action (too much) or allowing catastrophic autonomous decisions (too little).

Six HITL Patterns

Choosing the Right Human Checkpoint

01

Approval Gate

Agent proposes, human approves before execution. Hard separation between “suggest” and “do”. Best for high-risk operations such as database writes and financial transactions.

AcmeCorp Agent generates a DELETE query for old records — waits for DBA approval before executing.
02

Dual Approval (Four-Eyes Principle)

Two separate approvals required. Used in regulated industries (banking, healthcare). Best for irreversible, high-impact operations.

AcmeCorp Schema migration affecting production tables needs both the Data Architect and DBA sign-off.
03

Sampled Review

100% approval for high-risk, 5–20% random sampling for low-risk. Balances safety with throughput. Best for high-volume operations with varying risk.

AcmeCorp All DDL queries reviewed; SELECT queries spot-checked at 10% rate.
04

Exception-Based Review

Auto-proceed unless a policy trigger fires: low confidence, sensitive data detected, anomaly, cost threshold. Best for mature agents with well-defined policies.

AcmeCorp Standard dashboard queries auto-execute; queries touching PII columns pause for review.
05

Confidence-Based Escalation

Agent reports confidence level. Below threshold → pause and escalate. Above threshold → proceed autonomously. Best for agents that can self-assess uncertainty.

AcmeCorp Agent is 95% confident about revenue query → executes. 60% confident about a complex join → asks analyst.
06

Return of Control (ROC)

Goes beyond approve/reject. Human can modify parameters, redirect, or take over entirely. Best for collaborative workflows where humans and agents co-create.

AcmeCorp Agent drafts SQL, analyst tweaks the WHERE clause, agent executes the modified query.
Graduated Autonomy

From Full Oversight to Full Autonomy

Level Description Human Role AcmeCorp Use Case
1 Agent suggests, human approves everything Approver New deployment, first week
2 Agent acts on routine, pauses for high-stakes Reviewer Established agent, month 2+
3 Agent operates independently, humans review after Auditor Mature agent, known patterns
4 Fully autonomous with exception-based oversight Observer Low-risk dashboards, high-volume
Insight
Most production agents start at Level 1 and graduate to Level 2–3 over weeks/months as trust builds. Level 4 is rare and reserved for well-understood, low-risk operations.

Insert a human checkpoint whenever the agent encounters one of these situations:

  • Irreversible decisions — Database deletes, financial transfers, permanent record changes
  • Regulatory implications — Compliance reports, audit submissions, data governance actions
  • Empathy-requiring tasks — Customer complaints, sensitive communications, HR-related data
  • First-time actions — Agent has never performed this specific operation before
  • Cost thresholds exceeded — LLM spend beyond budget, compute costs beyond limits
