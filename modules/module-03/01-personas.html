DATA SUPPLY CHAIN — PERSONA MAP
The Scene

One Agent, Five Frustrated Users

AcmeCorp’s generic agent is overwhelmed. The data engineer needs SQL generation, the product owner just wants charts, and the steward is furious about unauthorized schema changes. Time to specialize.

AcmeCorp Scenario
Three tickets hit the backlog in one morning: “Agent generated a DROP TABLE statement,” “Agent showed raw customer emails to the VP of Sales,” and “Agent can’t find the new Shopify tables.” A single persona cannot satisfy all five roles. You need to split it up.
The Five Personas

Meet the Data Team

1 Data Engineer

Builds pipelines and transforms. The construction foreman — builds the structure that everything else stands on.

“The construction foreman — builds the structure.”
Schema Discovery SQL Generation Pipeline Execution
AcmeCorp in Action
AcmeCorp’s engineer agent automates their nightly ETL from Shopify into the data warehouse, writes dbt transformations, and monitors pipeline health.
2 Data Steward

Ensures quality and compliance. The gatekeeper who makes sure data stays clean, classified, and compliant.

“The quality inspector — checks that everything meets standards.”
Schema Comparison Quality Checks Access Control
AcmeCorp in Action
When AcmeCorp adds a new ‘loyalty_points’ column, the steward agent validates naming conventions, checks for PII, and logs the change.
3 Data Architect

Designs schema and governance. The strategic thinker who plans the data landscape for scale and clarity.

“The city planner — designs layouts for future growth.”
Schema Design Relationship Mapping Governance Templates
AcmeCorp in Action
AcmeCorp’s architect agent designed the star schema for their new product analytics warehouse — fact_sales, dim_products, dim_stores, dim_dates.
4 Quality Analyst

Validates and tests data accuracy. The investigator who hunts anomalies, runs checks, and sounds the alarm.

“The laboratory tester — runs experiments, flags anomalies.”
Validation Queries Anomaly Detection Report Generation
AcmeCorp in Action
Every morning, AcmeCorp’s QA agent runs 47 validation checks: NULL rates, value ranges, freshness, row counts. If anomalies spike, it alerts the team.
5 Product Owner

Defines metrics and business rules. The translator between business questions and data answers.

“The product manager — translates business needs to data.”
NL Queries Visualization Trend Analysis
AcmeCorp in Action
AcmeCorp’s VP of Sales uses the PO agent: “Show me same-store sales growth vs. last year, exclude newly opened stores.” The agent knows exactly which metric definitions to apply.
Why It Matters

Four Benefits of Persona-Based Design

1 Role-Based Access

Each persona only has access to the tools and data it needs. The Product Owner never sees raw SQL; the Engineer never touches business metric definitions. Security by design.

2 Easier Testing

Testing a focused agent is simpler than testing one that does everything. You can write targeted test suites per persona: “Does the QA agent correctly detect NULL spikes?” instead of testing thousands of edge cases at once.

3 Natural Team Fit

Each team member gets an agent that speaks their language. Engineers get technical output with schema details. Product owners get charts and summaries. No one wades through irrelevant information.

4 Clearer Validation

When something goes wrong, you know which persona was responsible. Debugging “the QA agent flagged a false positive” is far easier than debugging “the agent did something weird.”

A single “do everything” agent sounds appealing but creates three critical problems in production:

1. Blast radius. One misconfigured tool or permission gap affects every user. With personas, a bug in the Engineer agent doesn’t impact the Product Owner.

2. Prompt bloat. Stuffing 5 roles worth of instructions into a single system prompt degrades reasoning quality. Each persona gets a focused, optimized prompt.

3. Audit nightmares. When compliance asks “who accessed customer PII last Tuesday?”, you need clear role boundaries — not a monolithic log of everything the agent ever did.
