YOU WILL DECIDE

Real decisions. Real consequences.

The simulations don't hand you the answer. You'll face the same trade-offs engineers hit in production — and see what happens when you get it wrong.

⚡ DECISION
"Should the agent have direct, unsupervised access to the production database?"
Grant full access and it answers faster — but one bad query crashes the BI dashboard for all 200 stores. Restrict it and you need a validated query layer. You'll build both and choose.
→ Covered in Module 01 · Architecture Foundation
⚡ DECISION
"A regional manager asks the agent for employee salary data. What happens?"
The data exists in the warehouse. The agent can technically retrieve it. But salary is restricted to HQ Admin only. You'll design the RBAC guardrail that blocks this — and the audit trail that proves it did.
→ Covered in Module 03 · Persona Design & Guardrails
⚡ DECISION
"A new model version improves accuracy by 8% but increases cost by 340%. Do you ship it?"
You'll run the cost-performance matrix, calculate the Unreliability Tax, and decide whether to route to the cheaper model, the smarter model, or both — depending on query complexity.
→ Covered in Module 09 · Cost Optimization & Economics
⚡ DECISION
"Your eval gate shows a 6% drop in SQL accuracy after a dependency update. Ship or halt?"
The business is waiting for the release. But 6% wrong answers at scale means hundreds of bad decisions per day. You'll configure the gate threshold, write the rollback plan, and decide.
→ Covered in Module 05 · AgentOps: CI/CD & Eval Gates
⚡ DECISION
"The agent's context window is filling up mid-session. What gets kept — and what gets dropped?"
Drop the wrong thing and the agent loses track of the conversation. Keep too much and you blow the token budget. You'll implement a compression strategy and test it against AcmeCorp's longest queries.
→ Covered in Module 06 · Context Engineering & RAG
⚡ DECISION
"A user pastes a crafted prompt designed to override the agent's system instructions. Does it work?"
You'll run a prompt injection attack against your own agent in simulation, watch it fail (or succeed), then implement the defense layer — input sanitization, instruction hierarchy, and detection logging.
→ Covered in Module 08 · Security, Privacy & Compliance
